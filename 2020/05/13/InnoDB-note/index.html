MySQL InnoDB 引擎

MySQL

MySQL大致是三个部分： 连接客户端的各种连接器，数据存储和日志，MySQL服务

1 客户端，是各种编程语言的connector
2 MySQL服务，内部包含各种组件，实现各种功能
3 文件系统，数据存储与日志

InnoDB

InnoDB用户无法手动创建哈希索引，InnoDB会自调优(self-tuning)，判断合适就会建立相关哈希索引。

InnoDB支持事务、行锁、聚集索引和 MVCC 特性。

InnoDB整体也分为三层：
（1）内存结构(In-Memory Structure)，这一层在MySQL服务进程内
（2）OS Cache，这一层属于内核态内存
（3）磁盘结构(On-Disk Structure)，这一层在文件系统上

InnoDB内存结构包含四大核心组件：

  • 缓冲池(Buffer Pool)

    • 提升InnoDB性能,加速读请求，避免每次都访问磁盘。
    • InnoDB的缓冲池缓存表数据与索引数据,减少磁盘IO操作。

  • 写缓冲(Change Buffer)

    • 提升InnoDB性能,加速写请求，避免每次都访问磁盘。
    • 写的数据在缓冲页中：
      • 直接修改缓冲池中的页，一次内存操作；接着再写入 redo log，一次磁盘顺序写操作。
      • 读取，会命中缓冲池的页。缓冲池LRU数据淘汰，会将没用的页刷回磁盘。
      • 缓冲池的页，会定期刷到磁盘，批量写到磁盘。
    • 写的数据不在缓冲页中：
      • 不在缓冲池中操作，而是记录到写缓冲记录中。
      • 同时写入 redo log，一次磁盘顺序写操作。
      • 数据库异常崩溃，从 redo log 中恢复数据。
      • 写缓冲定期刷

  • 自适应哈希索引(Adaptive Hash Index)

    • 目的是提升InnoDB性能，加速读请求，减少索引查询的寻路路径。

  • 日志缓冲(Log Buffer)

    • 极大优化redo日志性能，并提供了高并发与强一致性的折衷方案。

插入方式

一系列的插入SQL语句统称为范插入语句。不同方式对自增键的影响不一样。

简单插入 simple insert

可以提前知道插入行数的普通insert/replace 语句，不论是单条还是多条。

大批量插入 bulk insert

不可以提前知道插入行数的insert 语句，就是批量插入。

INSERT INTO t1 (name,age) SELECT name,age FROM t2;
REPLACE … SELECT …;
LOAD DATA

混合插入 mixed-mode insert

有的行指定了自增键，不用数据库生成。有的行没有指定自增键，需要数据库生成。
INSERT INTO t1 (id,name) VALUES (111,”111”),(NULL, “abc”),(222,”222”),(NULL,”xyz”);

INSERT … ON DUPLICATE KEY UPDATE … ;
